# 启动说明

## 开发环境
```powershell
cd apps/api
uv run uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

## 生产环境
建议优先使用环境变量，不要把真实密钥和密码写入 `config.toml`。

### 1. 设置环境变量
```powershell
cd apps/api

$env:APP_LLM_BASE_URL="https://your-llm-provider.example.com/v1"
$env:APP_LLM_API_KEY="your-real-llm-api-key"
$env:APP_LLM_MODEL="gemini-2.5-pro"
$env:APP_LLM_FAST_MODEL="gemini-2.5-flash"
$env:APP_LLM_TIMEOUT_SECONDS="30"

$env:APP_DB_URL="postgresql://app_user:strong_password@127.0.0.1:5432/solohandle"

# 只有浏览器端需要 CORS；原生 App 直连时可以留空
$env:APP_CORS_ALLOW_ORIGINS="https://your-web-domain.com,https://admin.your-web-domain.com"

# 公网部署建议开启
$env:APP_API_TOKEN="replace-with-a-long-random-token"
```

### 2. 或者使用 `.env`
后端现在会自动读取 `apps/api/.env`。系统环境变量优先，`.env` 只作为兜底。

```dotenv
APP_LLM_BASE_URL=https://your-llm-provider.example.com/v1
APP_LLM_API_KEY=your-real-llm-api-key
APP_LLM_MODEL=gemini-2.5-pro
APP_LLM_FAST_MODEL=gemini-2.5-flash
APP_LLM_TIMEOUT_SECONDS=30
APP_DB_URL=postgresql://app_user:strong_password@127.0.0.1:5432/solohandle
APP_CORS_ALLOW_ORIGINS=https://your-web-domain.com,https://admin.your-web-domain.com
APP_API_TOKEN=replace-with-a-long-random-token
```

### 3. 启动服务
```powershell
cd apps/api/src
uv run uvicorn api.main:app --host 0.0.0.0 --port 8000
```

## Bearer Token 调用方式
如果设置了 `APP_API_TOKEN`，所有接口（`/router/health` 除外）都需要带上：
```http
Authorization: Bearer your-token
```

## Docker 启动数据库
在仓库根目录执行：
```powershell
powershell -ExecutionPolicy Bypass -File docker/start-db.ps1
```

## Docker 停止数据库
在仓库根目录执行：
```powershell
powershell -ExecutionPolicy Bypass -File docker/stop-db.ps1
```

## 上线前检查
1. 确认 `config.toml` 中不再包含真实密钥
2. 确认 PostgreSQL 密码不是默认值
3. 确认已设置 `APP_API_TOKEN`
4. 确认已通过反向代理提供 HTTPS
5. 确认移动端已同步新的 Bearer Token
